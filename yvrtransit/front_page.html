yvr transit data index

what's this?

This is an archive of "real-time" transit information from the TransLink GTFS-realtime Open API. It provides historical data of the location of every tracked vehicle in the TransLink service territory. It is updated every minute of every day.

The data are presented as they were received off the wire, as messages in the Google Protocol Buffers format, according to the GTFS-realtime documentation hosted by Google.

Each monthly rollup .zip contains daily archives in .tar.xz format for each feed type provided by TransLink.

You will need static GTFS data matching the period of the real-time data you want to analyze in order to make sense of them. TransLink updates their schedules at least four times a year. Please note that they publish the new static data before it takes effect and the static data sets are non-overlapping; you need to use the previous static data archive to interpret data from the end of the schedule period.
